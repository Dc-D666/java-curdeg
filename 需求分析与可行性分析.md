# 校园论坛系统 - 需求分析与可行性分析

## 一、项目概述

### 1.1 项目目标
基于课程提供的基础框架，开发一个功能完整的校园论坛系统，采用**前后端分离架构**：
- **前端**：JavaFX + FXML 桌面客户端
- **后端**：Spring Boot 3 + JPA 服务端

### 1.2 项目约束
- ✅ 必须复用课程提供的基础工程
- ✅ 前端禁止使用 Web 技术（Vue/React等）
- ✅ 后端禁止使用 MyBatis/MyBatis-Plus，必须使用 JPA
- ✅ 符合课程开发规范与评审标准

## 二、基础框架现状分析

### 2.1 前端框架（java-fx）现状

#### ✅ 已有的基础设施
1. **技术栈**

## 四、开发重点与难点分析

### 4.1 开发重点

#### P0 - 核心功能（必须完成）
1. **用户认证与权限控制**
   - 完善登录/注册/退出流程
   - 基于角色的界面动态显示
   - Token 过期自动重登录

2. **帖子与评论核心功能**
   - 帖子列表展示与分页
   - 帖子发布/编辑/删除
   - 评论发布/编辑/删除

3. **板块导航**
   - 板块列表展示
   - 按板块筛选帖子

#### P1 - 重要功能（应该完成）
1. **个人中心**
   - 个人信息编辑
   - 我的帖子/评论列表
   - 行为统计展示

2. **基础交互**
   - 点赞功能
   - 收藏功能
   - 搜索功能（关键词）

3. **管理员基础功能**
   - 帖子管理（删帖、置顶、加精）
   - 用户列表查看

#### P2 - 特色功能（可选，加分项）
1. **AI 集成**
   - 帖子/评论 AI 总结
   - AI 内容审核
   - 自然语言搜索 QA

2. **高级功能**
   - 消息通知
   - 举报处理流程
   - 数据统计面板

### 4.2 开发难点

#### 难点 1：JavaFX 复杂界面开发
**问题**：JavaFX 相比 Web 技术，UI 组件库较少，复杂交互实现难度大
**应对方案**：
- 复用课程框架中的基础组件（表格、对话框等）
- 参考现有 Controller 的实现模式
- 保持界面简洁，避免过度复杂的交互

#### 难点 2：权限控制的界面动态化
**问题**：不同角色看到的菜单、按钮、功能完全不同
**应对方案**：
- 在 `MainFrameController` 中根据用户角色动态加载菜单
- 在各个面板中根据角色控制按钮的可见性和可用性
- 后端做好接口权限校验，前端仅做展示控制

#### 难点 3：AI 功能集成
**问题**：需要对接大模型 API，处理异步请求和错误
**应对方案**：
- 设计合理的 AI 服务接口，封装大模型调用
- 采用异步处理，避免阻塞 UI
- 提供降级方案（AI 不可用时使用传统方式）

#### 难点 4：内容审核机制
**问题**：如何实现有效的违禁词过滤和违规内容处理
**应对方案**：
- 先实现简单的关键词过滤
- 后续可接入 AI 审核作为增强
- 设计举报流程，人工审核作为最终保障

## 六、开发建议

### 6.1 开发阶段划分

**第一阶段（核心功能）**
1. 前端对接登录、注册、退出
2. 前端实现帖子列表、详情、发布
3. 前端实现评论功能
4. 权限控制基础

**第二阶段（完善功能）**
1. 个人中心
2. 点赞、收藏
3. 搜索功能
4. 管理员基础功能

**第三阶段（特色功能）**
1. AI 功能集成
2. 消息通知
3. 举报流程
4. 数据统计

### 6.2 复用建议
- **前端**：大量复用 `LoginController`、`MainFrameController`、`HttpRequestUtil` 的代码模式
- **后端**：复用 `BbsPostService`、`BbsUserService` 的服务层架构
- **数据库**：在现有表结构基础上扩展，避免大改

### 6.3 风险控制
- 优先保证核心功能完成，特色功能可后续迭代
- 界面设计保持简洁，避免过度复杂
- AI 功能作为可选加分项，不影响核心流程

本项目基于完善的课程基础框架，需求明确，技术栈成熟，**整体可行性很高**。
