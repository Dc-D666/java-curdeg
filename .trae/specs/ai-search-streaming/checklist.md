# AI搜索流式输出 - Verification Checklist

## 后端验证
- [ ] SSE 接口能够正常访问
- [ ] SSE 事件格式符合 spec.md 中的定义
- [ ] 流式响应解析正确，能正确处理 OpenAI 兼容格式
- [ ] 能够正确处理 AI API 的流式输出和 `[DONE]` 标记
- [ ] posts 事件优先于 content 事件发送
- [ ] done 事件在流式完成时正确发送
- [ ] error 事件在异常时正确发送
- [ ] 原有的非流式接口仍然正常工作（向后兼容）
- [ ] 错误处理机制完善
- [ ] 超时设置合理

## 前端验证
- [ ] 能够建立 SSE 连接并接收数据
- [ ] 能够正确解析不同类型的 SSE 事件
- [ ] UI 能够随着流式数据更新
- [ ] 缓冲策略正常工作（不会每一个字都刷新 UI）
- [ ] 更新流畅，没有过度闪烁
- [ ] Markdown 格式能够正确渐进式渲染（加粗、斜体、删除线、红色标记）
- [ ] 相关帖子列表在 AI 开始生成之前或同时显示
- [ ] 复制答案功能在流式过程中也能正常工作
- [ ] 加载状态友好清晰，能够区分不同阶段
- [ ] 错误提示友好清晰
- [ ] 错误后可以重新搜索
- [ ] 降级机制正常工作（流式失败时可以回退到非流式）

## 整体验证
- [ ] 首字响应时间 < 3秒
- [ ] 完整流程能够正常运行
- [ ] 用户体验良好
- [ ] 没有明显的性能退化
- [ ] 多次连续搜索测试通过，没有状态残留
- [ ] 内存占用正常，长时间流式不会导致 OOM
