# 认证功能改进 - 产品需求文档

## Overview
- **Summary**: 修复注册账号的乐观锁异常，开发"忘记密码"功能，美化登录/注册/忘记密码三个认证页面的 UI。
- **Purpose**: 确保注册流程稳定性，提供密码自助找回能力，提升认证页面的视觉体验。
- **Target Users**: 所有使用学生交流社区认证功能的用户。

## Why
1. 注册账号时 JPA `@Version` 乐观锁机制在 `saveAndFlush` 调用时产生冲突异常，影响新用户注册成功率和体验
2. 用户忘记密码后无自助找回途径，只能依赖管理员手动重置，运维成本高
3. 当前登录/注册页面风格简陋（GridPane 老式布局），缺乏统一的 UI 设计语言

## Goals
- 修复注册账号乐观锁异常，确保新用户能够正常注册
- 实现"忘记密码"功能：输入邮箱 → 发送验证码 → 输入新密码 → 重置成功
- 统一美化登录页、注册页、忘记密码页的 UI（采用现代化设计风格）
- 三个页面保持一致的设计语言和交互体验

## Non-Goals (Out of Scope)
- 不修改后端积分/等级的初始化逻辑（已有正确实现）
- 不实现邮箱修改功能
- 不实现双因子认证
- 不修改 main-frame.fxml 或其他页面

## What Changes
- **修复**: `AuthService.registerUser()` 使用 `persist` 替代 `merge`，避免 `@Version` 乐观锁冲突
- **新增**: 后端 `POST /api/auth/resetPassword` 接口（验证邮箱验证码 + 修改密码）
- **新增**: 前端 `forgot-password.fxml` 忘记密码页面 + `ForgotPasswordController`
- **修改**: `login-view.fxml` 登录页面 UI 美化 + 添加"忘记密码"链接
- **修改**: `register-view.fxml` 注册页面 UI 美化（与登录/忘记密码统一风格）
- **修改**: `LoginController.java` 添加跳转到忘记密码页面的方法

## Impact
- Affected specs: `email-verification-registration`（复用已有验证码发送接口）
- Affected code:
  - 后端：`AuthService.java`, `AuthController.java`（新增重置密码）
  - 前端：`login-view.fxml`, `register-view.fxml`, `LoginController.java`, `RegisterController.java`
  - 新增：`forgot-password.fxml`, `ForgotPasswordController.java`

## ADDED Requirements

### Requirement: 注册账号乐观锁修复
系统 SHALL 在注册新用户时使用 JPA `persist` 操作，确保新实体的 `@Version` 字段初始值 0 被正确写入数据库，避免 `merge` 操作引发的乐观锁冲突。

#### Scenario: 新用户注册成功
- **GIVEN** 用户填写了合法的学号、昵称、邮箱、验证码、密码
- **WHEN** 点击"注册"按钮
- **THEN** 系统创建 Person 和 User 实体并持久化，返回"注册成功"，version 字段初始值为 0，无乐观锁异常

#### Scenario: 注册后自动登录成功
- **GIVEN** 用户刚刚注册成功
- **WHEN** 前端自动调用登录接口
- **THEN** 登录成功，`loginCount` 更新为 1，version 字段更新为 1，无冲突

### Requirement: 忘记密码功能
系统 SHALL 提供"忘记密码"功能，允许已注册用户通过邮箱验证码重置密码。

#### Scenario: 进入忘记密码页面
- **GIVEN** 用户在登录页面
- **WHEN** 点击"忘记密码"链接
- **THEN** 导航到忘记密码页面，页面包含邮箱输入框、验证码输入框、新密码输入框、确认密码输入框

#### Scenario: 发送重置密码验证码
- **GIVEN** 用户在忘记密码页面输入了已注册的邮箱
- **WHEN** 点击"发送验证码"按钮
- **THEN** 系统向该邮箱发送6位数字验证码，按钮进入60秒倒计时，验证类型为 "RESET_PASSWORD"

#### Scenario: 密码重置成功
- **GIVEN** 用户输入了邮箱、正确的验证码、合法的新密码
- **WHEN** 点击"重置密码"按钮
- **THEN** 后端验证邮箱存在、验证码正确、新密码合法，更新密码并返回成功，前端跳转到登录页面

#### Scenario: 邮箱未注册
- **GIVEN** 用户输入了系统中不存在的邮箱
- **WHEN** 点击"发送验证码"或"重置密码"
- **THEN** 系统返回通用错误"该邮箱未注册"（仅重置时校验），发送验证码时若邮箱未注册也提示"发送失败"

#### Scenario: 密码重置后旧密码失效
- **GIVEN** 用户通过忘记密码功能重置了密码
- **WHEN** 用户使用旧密码登录
- **THEN** 登录失败，提示"用户名或密码错误"

### Requirement: 认证页面 UI 美化
系统 SHALL 提供统一、现代化的登录、注册、忘记密码三个页面 UI 设计。

#### Scenario: 登录页面美观一致
- **GIVEN** 用户打开登录页面
- **WHEN** 页面加载完成
- **THEN** 页面使用现代化卡片式布局，含 Logo、标题、输入框、按钮，底部有"忘记密码?"和"注册新账号"链接，整体居中，风格简洁大方

#### Scenario: 注册页面与登录统一
- **GIVEN** 用户打开注册页面
- **WHEN** 页面加载完成
- **THEN** 页面布局风格与登录页面完全一致，使用相同的配色方案、字体大小、卡片圆角、按钮样式

#### Scenario: 忘记密码页面与登录统一
- **GIVEN** 用户打开忘记密码页面
- **WHEN** 页面加载完成
- **THEN** 页面布局风格与登录页面完全一致，使用相同的配色方案、字体大小、卡片圆角、按钮样式

## MODIFIED Requirements

系统 SHALL 在美化注册页面 UI 时保持所有现有功能和字段不变（学号、昵称、邮箱、验证码、密码、重复密码），所有 fx:id 和事件绑定保持不变。

### Requirement: 登录页面现有功能保持不变
系统 SHALL 在美化登录页面 UI 时保持所有现有功能不变（记住密码、登录、跳转注册），所有 fx:id 和事件绑定保持不变。

## 业务规则

### 验证码规则
- 验证码长度：6位纯数字
- 验证码有效期：5分钟
- 发送间隔：同一邮箱至少60秒
- 验证类型新增：`RESET_PASSWORD`

### 密码规则
- 新密码长度至少6位
- 新密码不能与旧密码完全相同（建议但不强制）

### 邮箱规则
- 发送重置验证码时：即使邮箱未注册也显示"发送成功"（防枚举）
- 重置密码时：明确检查邮箱是否已注册

## 验收标准

### AC-1: 注册无乐观锁异常
- **Given**: 新用户信息完整合法
- **When**: 多次注册新账号
- **Then**: 每次都注册成功，无任何乐观锁异常日志
- **Verification**: `programmatic`

### AC-2: 忘记密码完整流程
- **Given**: 已注册用户 user1@test.com
- **When**: 用户进入忘记密码页 → 输入邮箱 → 点击发送验证码 → 输入验证码和新密码 → 点击重置
- **Then**: 密码重置成功，用新密码可登录，旧密码登录失败
- **Verification**: `programmatic`

### AC-3: 忘记密码防枚举
- **Given**: 未注册邮箱 fake@noexist.com
- **When**: 在忘记密码页面输入该邮箱并点击发送验证码
- **Then**: 不暴露邮箱未注册，返回"验证码已发送"（或静默失败）
- **Verification**: `programmatic`

### AC-4: 三页面 UI 统一
- **Given**: 登录页、注册页、忘记密码页
- **When**: 分别打开三个页面
- **Then**: 三页面使用相同的配色、布局结构、按钮样式、卡片风格
- **Verification**: `human-judgment`

### AC-5: 登录页面功能完整
- **Given**: 美化后的登录页
- **When**: 用户登录、点击注册、点击忘记密码、使用记住密码
- **Then**: 所有功能正常工作
- **Verification**: `programmatic`

### AC-6: 注册页面功能完整
- **Given**: 美化后的注册页
- **When**: 用户填写表单、发送验证码、注册
- **Then**: 所有功能正常工作，与修改前行为一致
- **Verification**: `programmatic`
