# 校园论坛私信功能 PRD

## Overview
- **Summary**: 为校园论坛系统实现完整的私信功能，包括会话列表、消息发送与接收、未读提醒、互关权限控制等。
- **Purpose**: 解决用户间的点对点沟通需求，提供安全、方便的私信功能，同时通过互关机制防止骚扰。
- **Target Users**: 所有登录的校园论坛用户（学生、教师、管理员）。

## Goals
1. 在个人中心添加"我的私信"入口，并展示未读消息红点提醒
2. 从用户主页可以直接发起私信会话
3. 实现互关权限控制：未互关时只能发1条消息，互关后无限制
4. 实现文字和图片消息的发送和展示
5. 提供消息历史记录查看功能
6. 确保功能不影响现有系统的其他功能

## Non-Goals (Out of Scope)
1. 群聊/多人会话功能
2. 消息撤回功能
3. 消息加密功能
4. 视频/语音通话功能

## Background & Context
- 现有系统已有用户关注功能（`BbsFollow`表），可以判断用户间的关注关系
- 已有个人中心页面框架，可以在菜单中新增选项
- 已有用户主页功能，可在现有按钮基础上增加私信功能入口
- 系统采用Spring Boot后端 + JavaFX前端的架构

## Functional Requirements

### FR-1: 数据库表设计
- 创建私信会话表（`bbs_conversation`）：存储两个用户之间的会话关系
- 创建私信消息表（`bbs_message`）：存储具体的私信消息

### FR-2: 后端API接口
- 创建会话接口：从用户主页发起新会话
- 发送消息接口：支持文字和图片
- 获取会话列表接口：按时间倒序，显示最近消息
- 获取消息历史接口：获取某个会话的完整历史
- 标记已读接口：打开会话后标记消息为已读
- 获取未读总数接口：用于菜单红点显示

### FR-3: 前端菜单和未读提醒
- 在个人中心菜单中添加"我的私信"选项
- 在"我的私信"菜单旁显示未读消息数量的红点提示
- 实现动态更新未读消息数

### FR-4: 私信会话列表页面
- 显示用户的所有会话
- 每个会话显示对方头像、昵称、最后一条消息预览
- 点击会话进入聊天页面
- 显示未读消息提示

### FR-5: 私信聊天页面
- 显示历史消息列表
- 支持发送文字消息
- 支持发送图片消息
- 区分已读和未读消息
- 互关状态下的消息发送限制提示

### FR-6: 用户主页私信功能
- 只在已关注对方时显示"私信"按钮
- 点击"私信"按钮直接打开与该用户的会话
- 如果没有会话则创建新会话

### FR-7: 互关权限控制
- 检查双方是否互相关注
- 未互关时，一方只能发送1条消息给对方
- 互关后，双方可以无限制发送消息
- 在发送消息时进行验证并给出相应提示

## Non-Functional Requirements

### NFR-1: 性能要求
- 会话列表加载时间小于1秒
- 消息发送响应时间小于200ms
- 未读消息数实时更新（轮询或其他方式）

### NFR-2: 可用性要求
- 界面清晰，使用简单
- 消息显示流畅，支持长文本
- 支持图片查看和预览

### NFR-3: 安全性要求
- 只能访问自己的会话和消息
- 防止XSS攻击（消息内容转义）
- 图片上传时进行安全验证

## Constraints

### Technical
- 后端使用Spring Boot + JPA + MySQL
- 前端使用JavaFX
- 数据库表名遵循现有命名规范（`bbs_`前缀）
- API路径遵循现有规范（`/api/bbs/`前缀）

### Business
- 数据库修改需要先提供SQL脚本
- 不能破坏现有功能的正常运行
- 需要兼容现有用户数据

## Assumptions
1. 现有用户表（`sys_user`）和关注表（`bbs_follow`）可以正常使用
2. 现有的文件上传功能可以复用（用于消息图片）
3. 用户已有JWT认证，身份验证已完善
4. 前端可以通过API访问后端服务

## Acceptance Criteria

### AC-1: 数据库表创建成功
- **Given**: 数据库已连接
- **When**: 执行私信功能的SQL脚本
- **Then**: `bbs_conversation`和`bbs_message`表创建成功，索引正确
- **Verification**: programmatic
- **Notes**: 验证SQL执行成功，表结构完整

### AC-2: 菜单配置与未读提醒
- **Given**: 用户已登录
- **When**: 访问个人中心
- **Then**: 可以看到"我的私信"菜单项；如有未读消息，显示红色数字提醒
- **Verification**: human-judgment

### AC-3: 用户主页私信功能
- **Given**: 访问用户A的主页
- **When**: 当前用户未关注用户A
- **Then**: 不显示"私信"按钮；当前用户已关注用户A时，显示绿色"私信"按钮
- **Verification**: human-judgment

### AC-4: 会话列表功能
- **Given**: 用户进入"我的私信"
- **When**: 会话列表加载完成
- **Then**: 显示所有会话，按时间倒序，有未读的在前
- **Verification**: human-judgment

### AC-5: 消息发送与接收
- **Given**: 用户打开与某个用户的会话
- **When**: 发送文字或图片消息
- **Then**: 消息成功显示在聊天列表中，对方可以收到并回复
- **Verification**: human-judgment

### AC-6: 互关权限控制
- **Given**: 用户A与用户B的关系
- **When**: 情况1：A已关注B，但B未关注A；情况2：两人互关
- **Then**: 情况1：A只能发送1条消息给B；情况2：无限制发送
- **Verification**: human-judgment

### AC-7: 未读消息标记
- **Given**: 有未读消息的会话
- **When**: 用户打开会话
- **Then**: 该会话的未读消息标记为已读，菜单红点相应减少
- **Verification**: human-judgment
