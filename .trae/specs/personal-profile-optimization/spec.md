# 个人资料页面优化 - Product Requirement Document

## Overview
- **Summary**: 优化现有个人资料页面的布局、交互体验和功能，解决当前页面存在的信息杂乱、操作不便、缺乏分组等问题。
- **Purpose**: 提升用户在编辑和查看个人资料时的体验，使页面更加清晰、易用、美观，同时增强功能的完整性和可靠性。
- **Target Users**: 校园论坛系统的所有用户（学生、管理员、超级管理员）

## Goals
- [ ] 优化页面布局，使用分组卡片设计，提高信息可读性
- [ ] 改进头像管理，添加图片上传和预览功能
- [ ] 优化隐私设置的展示方式
- [ ] 添加日期选择器组件，提升生日输入体验
- [ ] 添加表单验证，提高数据质量
- [ ] 优化编辑和查看模式的切换体验
- [ ] 改进统计信息展示，使其更加突出

## Non-Goals (Out of Scope)
- [ ] 不修改后端API接口（除非必要）
- [ ] 不重构整个用户系统架构
- [ ] 不添加个人资料以外的新功能

## Background & Context
当前个人资料页面存在以下问题：
1. 所有字段堆积在一个GridPane中，缺乏视觉分组，信息密度过高
2. 头像管理只有URL输入框，没有上传按钮和预览功能，用户体验差
3. 隐私设置在查看模式下也显示，视觉上不够清晰
4. 生日使用普通TextField输入，没有日期选择器，体验不好
5. 缺乏输入验证，容易导致无效数据
6. 统计信息位置不够突出，且点击数、关注数、粉丝数等卡片缺乏交互
7. 编辑按钮在页面最底部，需要滚动才能看到
8. 保存/取消按钮在编辑模式下才显示，缺乏过渡动画
9. 字段标签不够清晰，部分标签（如"学院"）可能需要更明确的说明

## Functional Requirements
- **FR-1**: 页面布局重构，使用分组卡片设计（基本信息、详细信息、隐私设置、统计信息）
- **FR-2**: 头像管理功能优化，添加图片上传按钮和预览功能
- **FR-3**: 隐私设置展示优化，查看模式下简化显示，编辑模式下显示完整设置
- **FR-4**: 添加日期选择器组件用于生日输入
- **FR-5**: 添加表单验证功能（昵称长度限制、邮箱格式验证、电话格式验证等）
- **FR-6**: 统计信息卡片优化，添加交互效果（悬停、点击跳转）
- **FR-7**: 编辑按钮位置优化，放置在更醒目的位置
- **FR-8**: 添加编辑/查看模式切换的过渡效果

## Non-Functional Requirements
- **NFR-1**: 页面响应时间应在1秒以内
- **NFR-2**: 表单验证错误提示应清晰明确
- **NFR-3**: 页面布局应适应不同分辨率的屏幕
- **NFR-4**: 代码应保持可维护性和可扩展性

## Constraints
- **Technical**: 必须使用JavaFX 21框架，保持与现有代码风格一致
- **Business**: 功能修改应不影响现有数据结构
- **Dependencies**: 依赖现有的后端API和用户模型

## Assumptions
- [ ] 后端API保持不变，只修改前端实现
- [ ] 现有的用户模型数据结构足够支持优化后的功能
- [ ] 文件上传功能已有后端支持

## Acceptance Criteria

### AC-1: 分组卡片布局
- **Given**: 用户打开个人资料页面
- **When**: 页面加载完成
- **Then**: 页面显示为清晰的分组卡片布局（基本信息、详细信息、隐私设置、统计信息），视觉层次清晰
- **Verification**: `human-judgment`
- **Notes**: 每个分组应有明确的标题和分隔

### AC-2: 头像上传功能
- **Given**: 用户在编辑个人资料页面
- **When**: 用户点击头像或上传按钮
- **Then**: 弹出文件选择对话框，支持选择图片文件，选择后图片立即预览显示
- **Verification**: `programmatic`

### AC-3: 隐私设置显示优化
- **Given**: 用户在查看模式
- **When**: 查看个人资料
- **Then**: 隐私设置简化显示（如只显示小图标或标签），编辑模式下显示完整的下拉框
- **Verification**: `human-judgment`

### AC-4: 日期选择器
- **Given**: 用户在编辑个人资料页面
- **When**: 用户点击生日输入框
- **Then**: 弹出日期选择器组件，用户可选择日期，选择后自动填入输入框
- **Verification**: `programmatic`

### AC-5: 表单验证
- **Given**: 用户在编辑个人资料页面
- **When**: 用户输入无效数据（如过长的昵称、无效的邮箱格式）并尝试保存
- **Then**: 显示清晰的错误提示，阻止保存，直到所有字段验证通过
- **Verification**: `programmatic`

### AC-6: 统计卡片交互
- **Given**: 用户在个人资料页面
- **When**: 用户点击统计卡片（发帖数、关注数、粉丝数）
- **Then**: 跳转到相应的页面（我的帖子、我的关注、我的粉丝）
- **Verification**: `programmatic`

### AC-7: 编辑按钮位置优化
- **Given**: 用户在查看模式
- **When**: 页面加载完成
- **Then**: 编辑按钮位于页面右上角或统计信息附近，醒目易见
- **Verification**: `human-judgment`
