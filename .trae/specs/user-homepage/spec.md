# 用户主页 - 产品需求文档

## 概述
- **Summary**: 设计并实现一个用户主页功能，允许用户在帖子详情页点击其他用户头像后跳转到该用户的个人主页，查看其公开信息和动态。
- **Purpose**: 增强用户之间的社交互动，提供更全面的用户信息展示，改善用户体验。
- **Target Users**: 所有论坛用户（学生、管理员等）。

## 目标
1. 提供用户公开信息展示（头像、昵称、个性签名等）。
2. 展示用户发帖列表（审核通过的帖子）。
3. 提供关注/取消关注功能。
4. 展示用户数据统计（发帖数、粉丝数、关注数等）。
5. 保持与现有UI风格一致。

## 非目标（不包含）
1. 不修改用户自己的个人中心页面。
2. 不提供编辑其他用户信息的功能。
3. 不实现私信功能（但保留UI入口）。
4. 不包含用户评价功能。

## 背景与上下文
现有系统已经有个人中心页面（用户查看自己的信息），但缺少用户主页（查看其他用户信息）。在帖子详情页用户头像可以看到，但点击后没有任何交互。

### 技术约束
- 使用JavaFX前端
- 保持与现有设计风格一致（蓝色主题色#4a90e2）
- 复用现有的后端API
- 使用现有的数据模型

## 功能需求
### FR1: 用户基本信息展示
- 展示用户头像
- 展示用户昵称
- 展示用户个性签名
- 展示用户加入时间（可选）

### FR2: 用户数据统计
- 展示发帖数
- 展示粉丝数（关注者数）
- 展示关注数
- 展示获赞数（可选）

### FR3: 关注/取消关注功能
- 提供关注按钮
- 点击后切换关注状态
- 显示当前关注状态（已关注/未关注）

### FR4: 用户发帖列表
- 展示该用户发布的帖子（仅审核通过的）
- 支持分页查看
- 点击帖子可以跳转到帖子详情
- 显示帖子标题、内容摘要、发布时间、点赞数、评论数

### FR5: 头像点击跳转
- 在帖子详情页点击作者头像跳转到用户主页
- 在评论区点击评论者头像跳转到用户主页

### FR6: 私信功能入口（预留）
- 提供私信按钮UI
- 点击后显示"功能开发中"提示
- 不实现具体的私信功能

## 非功能需求
### NFR1: 性能
- 页面加载时间不超过2秒
- 支持分页加载，避免一次性加载过多数据

### NFR2: 用户体验
- 界面简洁美观，与现有风格一致
- 操作流程清晰
- 提供友好的加载提示和错误提示

### NFR3: 可维护性
- 代码结构清晰
- 复用现有组件
- 遵循现有代码规范

## 约束
### 技术
- 技术栈：Java 21 + JavaFX 17 + Spring Boot 3.2.5
- 必须使用现有的API和数据模型
- 界面布局必须与现有风格保持一致

### 业务
- 只能查看公开信息
- 不能查看被禁言用户的帖子（或显示特定提示）
- 必须符合用户隐私设置（如果有）

## 假设
1. 后端已经有获取用户公开信息的API
2. 后端已经有获取用户发帖列表的API
3. 后端已经有关注/取消关注的API
4. 用户隐私设置默认允许查看公开信息

## 接受标准

### AC1: 用户信息展示完整
- **Given**: 用户进入某个用户的主页
- **When**: 页面加载完成
- **Then**: 应该显示用户头像、昵称、个性签名、发帖数、粉丝数、关注数
- **Verification**: human-judgment
- **Notes**: 信息展示清晰，布局美观

### AC2: 关注功能正常工作
- **Given**: 当前用户已登录且查看的不是自己的主页
- **When**: 点击关注/取消关注按钮
- **Then**: 按钮状态更新，关注数同步更新
- **Verification**: programmatic

### AC3: 发帖列表展示正常
- **Given**: 用户主页已加载
- **When**: 查看发帖列表
- **Then**: 显示该用户的帖子（仅审核通过），支持分页，点击可跳转详情
- **Verification**: programmatic + human-judgment

### AC4: 头像点击跳转
- **Given**: 在帖子详情页或评论区
- **When**: 点击用户头像
- **Then**: 在新标签页打开该用户的主页
- **Verification**: programmatic

### AC5: 界面风格一致
- **Given**: 用户主页已加载
- **When**: 对比现有页面风格
- **Then**: 界面风格与现有页面保持一致（颜色、布局、字体等）
- **Verification**: human-judgment

### AC6: 处理异常情况
- **Given**: 网络请求失败或数据为空
- **When**: 加载用户主页
- **Then**: 显示友好的错误提示或空状态提示
- **Verification**: human-judgment

### AC7: 私信入口显示
- **Given**: 用户主页已加载且查看的不是自己的主页
- **When**: 查看用户信息区
- **Then**: 显示私信按钮，点击后提示"功能开发中"
- **Verification**: human-judgment

2. 是否需要显示用户的点赞记录？（暂时不需要）
3. 是否需要显示用户的评论列表？（暂时不需要）
