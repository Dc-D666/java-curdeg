# 帖子检索功能 - UI设计方案

## 一、整体布局

采用与PostList类似的BorderPane布局：
- **顶部**: 搜索模式选择 + 搜索输入区
- **中部**: 搜索结果表格
- **底部**: 分页控件

## 二、顶部搜索区设计

### 2.1 搜索模式选择
使用ToggleGroup + 三个ToggleButton来实现三种搜索模式的切换：

```
[标题检索] [全文检索] [AI检索]
```

- **样式**: 默认样式为普通按钮，选中状态为蓝色背景（#1890ff）+ 白色文字
- **默认选中**: "标题检索"
- **交互**: 点击切换模式，同一时间只有一个模式被选中

### 2.2 搜索输入区
- **搜索提示文本**:
  - 标题检索: "输入帖子标题关键词..."
  - 全文检索: "输入帖子内容关键词..."
  - AI检索: "用自然语言描述你想找的内容..."

- **搜索按钮**: 蓝色背景（#1890ff），白色文字，与现有searchButton样式一致

- **布局示例**:
```
[标题检索●] [全文检索 ] [AI检索 ]  [_________________________] [搜索]
```

### 2.3 AI检索特殊提示（可选）
当选择"AI检索"时，在输入框下方显示一行提示文字：
```
提示：AI检索会智能理解您的需求，例如"找关于Java学习的帖子"
```

## 三、搜索结果表格

与PostList保持一致的表格结构：

| 列名 | 宽度 | 说明 |
|------|------|------|
| 标题 | 300px | 帖子标题 |
| 作者 | 150px | 作者头像+昵称 |
| 发布时间 | 150px | yyyy-MM-dd HH:mm |
| 点赞数 | 80px | 数字 |
| 评论数 | 80px | 数字 |
| 状态 | 100px | 状态文字 |
| **匹配度** | 100px | **新增**，显示搜索匹配程度（仅AI检索时显示） |

## 四、分页区设计

与PostList保持一致：
- 显示"共 X 条，第 X 页"
- 每页显示条数选择（10/20/50）
- 上一页/下一页按钮

## 五、状态提示设计

### 5.1 搜索中状态
在搜索进行时，显示加载提示：
```
搜索中，请稍候...
```

### 5.2 无结果状态
当搜索无结果时，显示友好提示：
```
未找到匹配的帖子，试试其他关键词？
```

### 5.3 AI检索特殊状态（可选）
AI检索时可以显示：
```
AI正在为您智能匹配...
```

```xml
                    <ToggleButton text="AI检索" />

        </TableView>

        </HBox>

```

## 八、样式细节

### 8.1 按钮样式
```css
/* 搜索按钮（与现有一致） */
.search-button {
    -fx-background-color: #1890ff;
    -fx-text-fill: white;
}

/* 搜索模式切换按钮 - 选中状态 */
.toggle-button:selected {
    -fx-background-color: #1890ff;
    -fx-text-fill: white;
}
```

### 8.2 间距与边距
- 顶部区域: padding 10px
- 各控件间距: 10px
- 表格与其他区域间距: 10px

## 九、交互流程

1. **页面加载** → 显示默认搜索模式（标题检索）
2. **选择搜索模式** → 更新提示文本，显示/隐藏AI提示
3. **输入关键词** → 点击搜索或按回车
4. **搜索执行** → 显示加载状态 → 展示结果
5. **点击结果** → 打开帖子详情
6. **分页操作** → 更新结果

## 十、与PostList的关系

- PostList继续保留现有功能（板块筛选+简单搜索）
- 新的"帖子检索"专注于高级搜索功能
- 两者共享帖子详情打开逻辑
